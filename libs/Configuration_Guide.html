Similarity Extractor Configuration Guide

Welcome to the Similarity Extractor Configuration Guide!

This is the final section of the Similarity Extractor User Guide.
This will help you choose the best settings for your specific needs.
It will also help you understand how to use the Similarity Extractor program effectively by doing so.

Section 1: Non-algorithm specific settings

  1. The most important setting to configure is the difference toggle.
    1. This setting will determine whether the Similarity Extractor will extract the differences between the two audio files or the similarities.
    2. This is important because it will save you time and effort when you are trying to find the differences between two audio files.
    3. By default, it is set to "False", meaning that the Similarity Extractor will only extract the similarities.
    4. If you want to extract both the similarities and differences, simply click on the button to set it to "True".
  2. Another important setting to configure is the output file name.
    1. This setting will determine the name of the output file that is generated by the Similarity Extractor.
    2. This is important because it will help you identify the output file and make it easier to find later on.
    3. By default, it is set to "Optional"; setting it to this will make the Similarity Extractor use the name of the first audio file as the output base name.
    4. If you want to use a different name, simply type in the name you want to use.
      1. For example, if you typed in the name "Neo-Aspect", the output file names would be:
        1. Neo-Aspect_similarities.wav
        2. Neo-Aspect_differences_1.wav
        3. Neo-Aspect_differences_2.wav

Section 2: Bertom Phantom Centre

  1. There is only one setting unique to the Bertom Phantom Centre algorithm.
  2. This setting is the "Output Similarity of Inverted Difference" toggle.
    1. This setting will cause the Similarity Extractor to run the algorithm twice.
    2. Once with the original audio files and a second time with the difference outut of the first run as the input.
    3. For this pass one of the outputs will be phase inverted, this will allow you to see any similarities that were wrongly identified as differences.

Section 3: MDX23C

  1. Just like the Bertom Phantom Centre algorithm, there is only one setting for the MDX23C algorithm.
  2. This setting is the model section dropdown menu.
    1. This setting will determine which model the Similarity Extractor will use to extract the similarities.
    2. By default, it is set to "MDX23C - 2K FFT".
    3. If you want to use a different model, simply select the model you want to use from the dropdown menu.
    4. As of writing this guide, there are 2 models to choose from:
      1. MDX23C - 2K FFT
      2. MDX23C - 8K FFT
    5. If you are unsure which model to use, I would recommend using the "2K FFT" model.
    6. This model is the least likely to miss any similarities.
    7. However, the "8K FFT" model will provide cleaner results as it uses a higher FFT size which means it has a higher frequency resolution.
    8. But it will also take longer to process the audio files and use more resources.
    9. So if you are using a low-end computer, I would recommend using the "2K FFT" model.

Section 4: VR V6.0.0b4

  1. There are three settings inside the VR V6.0.0b4 algorithm.
  2. The first setting is the Model Selection dropdown menu.
    1. This setting will determine which model the Similarity Extractor will use to extract the similarities.
    2. By default, it is set to "VR V6.0.0b4 - 2K FFT".
    3. If you want to use a different model, simply select the model you want to use from the dropdown menu.
    4. As of writing this guide, there are 2 models to choose from:
      1. VR V6.0.0b4 - 2K FFT
      2. VR V6.0.0b4 - 4K FFT
    5. If you are unsure which model to use, I would recommend using the "4K FFT" model.
    6. However, it is the most resource intensive model and will take longer to process the audio files.
    7. So if you are using a low-end computer, I would recommend using the "2K FFT" model.
  3. The second setting is the Model Focus dropdown menu.
    1. This setting will determine which output the Similarity Extractor will priorities.
    2. By default, it is set to "Similarity".
    3. This means that the similarity output will be fuller at the cost of worse differences.
    4. If you wish to have fuller differences, simply select the "Difference" option from the dropdown menu.
    5. This will make the Similarity Extractor focus on the differences at the cost of the similarities.
  4. The final setting is the "Double Processing" toggle.
    1. This setting will determine whether the Similarity Extractor will run the algorithm twice.
    2. Once with the original audio files and a second time with the difference outut of the first run as the input.
    3. For this pass one of the outputs, this will allow for more aggressive processing of the audio files.
    4. By default, it is set to "False", meaning that the Similarity Extractor will only run the algorithm once.

Section 5: Post Processing Options

  1. There are four post processing options available.
  2. These options are the same as the models used in the MDX23C and VR V6.0.0b4 algorithms.
  3. These options are:
    1. MDX23C - 2K FFT
    2. MDX23C - 8K FFT
    3. VR V6.0.0b4 - 2K FFT
    4. VR V6.0.0b4 - 4K FFT
  4. These options will allow you to process the output of the Similarity Extractor with a different model.
  5. This is useful for cleaning up the output of the Similarity Extractor.
  6. By default, it is set to "False".
  7. The best model to use depends on the algorithm you used to extract the similarities.
  8. If you used the MDX23C algorithm, I would recommend using the model you didn't use to extract the similarities.
    1. For example, if you used the "MDX23C - 2K FFT" model, I would recommend using the "MDX23C - 8K FFT" model.
  9. If you used the VR V6.0.0b4 algorithm, I would recommend using the "MDX23C - 8K FFT" model.
    1. This model will provide the best results for cleaning up the output of the Similarity Extractor.
  10. However, if you are using the Bertom Phantom Centre algorithm, I would recommend not using post processing.
    1. This is because the Bertom Phantom Centre algorithm results do not lend themselves to effective post processing.
  11. However, if you are using a low-end computer, I wouldn't recommend using post processing.
    1. This is because the post processing will take a long time to process the audio files.
    2. And it will use a lot of resources.

Section 6: Final Notes

This section will mainly be used to provide you with some final notes on which settings to use.

  1. If you are using a low-end computer, or laptop, which does not have a dedicated GPU, I would recommend using the following settings:
    1. Bertom Phantom Centre
    2. Post Processing: False
  2. If you are using a mid-range computer, or laptop, which has a low-end dedicated GPU, I would recommend using the following settings:
    1. MDX23C - 2K FFT
    2. Post Processing: False
  3. But if you wish to have fuller differences, I would recommend using the following settings:
    1. VR V6.0.0b4 - 2K FFT
    2. Model Focus: Difference
    3. Double Processing: True
    4. Post Processing: False
  4. If you are using a mid-range computer, or laptop, which has a mid-range dedicated GPU, I would recommend using the following settings:
    1. MDX23C - 8K FFT
    2. Post Processing: False
  5. But if you wish to have fuller differences, I would recommend using the following settings:
    1. VR V6.0.0b4 - 4K FFT
    2. Model Focus: Difference
    3. Double Processing: True
    4. Post Processing: MDX23C - 2K FFT
  6. If you are using a high-end computer, or laptop, which has a high-end dedicated GPU, I would recommend using the following settings:
    1. MDX23C - 8K FFT
    2. Post Processing: MDX23C - 2K FFT
  7. But if you wish to have fuller differences, I would recommend using the following settings:
    1. VR V6.0.0b4 - 4K FFT
    2. Model Focus: Difference
    3. Double Processing: True
    4. Post Processing: MDX23C - 8K FFT
